# Инструкции для AI-ассистентов в проекте Driver95

Этот файл содержит правила и ограничения для последующей разработки и поддержки проекта Driver95. Пожалуйста, строго соблюдайте их при работе над кодом.

---

## 🚫 Жёсткие ограничения проекта

1. **Не изменять файлы логики и данных без явного запроса**:
   - `app.js` содержит ядро маршрутизации экранов, переводы и логику сессий.
   - `data.js` содержит полную базу вопросов на двух языках.
   - Любые изменения стиля или интерфейса должны производиться **исключительно** через `style.css` и добавление/изменение вспомогательных классов в `index.html`.

2. **Сохранение идентификаторов (IDs)**:
   - Все существующие ID in `index.html` завязаны на обработчики событий в `app.js`. Переименование или удаление элементов с этими ID приведет к поломке функционала.

3. **Отсутствие внешних ресурсов и шрифтов**:
   - Не подключайте внешние CSS-библиотеки (например, Tailwind, Bootstrap).
   - Не импортируйте шрифты Google Fonts или сторонние ассеты без согласования. Используйте системный стек шрифтов (Inter/San Francisco/Segoe UI).

---

## 🎨 Стандарты дизайна и интерфейса

1. **Цветовая схема (Dark Mode по умолчанию)**:
   - Фон приложения: `#060b13`
   - Фон экранов/карточек (`.phoneShell`): `#0d1527`
   - Внутренние блоки (кнопки, варианты): `#132038`
   - Основной синий: `#1d63ed`
   - Зеленый (успех): `#10b981`
   - Красный (ошибка): `#ef4444`

2. **Чекбоксы в Quiz**:
   - Стилизуются с помощью псевдоэлементов `.answer::before` с фоновым SVG для галочек/крестиков.
   - Избегайте скачков интерфейса (layout jumping). Все варианты ответов должны иметь `min-height: 56px` и фиксированный размер иконки.

3. **Скрытие пустых элементов**:
   - Элемент изображения `#questionImage` должен полностью скрываться (`display: none !important`), если он пустой (класс `.hidden`), чтобы не занимать пустое место над вопросом.

---

## 🔄 Процесс разработки

- Работа над интерфейсом ведется поэтапно.
- После завершения каждого этапа (например, рефакторинг экрана) необходимо останавливаться, демонстрировать diff-код пользователю и фиксировать коммитом перед переходом к следующей задаче.
