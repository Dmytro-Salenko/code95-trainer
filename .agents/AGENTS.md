## 🎯 Назначение, цели и философия проекта

**Driver95** — это современное, легкое и полностью автономное веб-приложение (Progressive Web App, PWA), предназначенное для подготовки водителей к государственному экзамену на **Код 95** в Польше (Kwalifikacja wstępna) в центрах WORD.

> Подробное видение проекта, его миссия, целевая аудитория и ценности описаны в документе [VISION.md](file:///Users/salenkodimitry/.gemini/antigravity/scratch/code95-trainer/VISION.md).

### Философия проекта:
1. **Offline-first**: Приложение должно функционировать без подключения к сети. Вопросы, переводы, изображения и логика прогресса хранятся локально в браузере.
2. **Мультиязычность**: Поддерживается подготовка на 8 языках, с особым акцентом на русский (`ru`), украинский (`uk`) и польский (`pl`) языки для русско- и украиноязычных водителей в Польше.
3. **Микроскопический размер и производительность**: Приложение написано на чистом HTML, Vanilla CSS & Vanilla JS для мгновенной загрузки на любых смартфонах.
4. **Конфиденциальность**: Персональные данные пользователя не собираются. Статистика хранится в браузере.

Приложение спроектировано как статическое одностраничное приложение (SPA) с сервис-воркером для оффлайн-кэширования.

### Структура каталогов:

## 📦 Основные компоненты и базы данных

### 1. Структура базы вопросов (`data.js`)
Данные инициализируются в глобальный объект `window.DRIVER95_DATA`. Вопросы распределены по языковым ключам (`de`, `en`, `ru`, `es`, `pl`, `it`, `tr`, `uk`). Каждый вопрос представляет собой объект:
```typescript
interface Question {
  id: number;              // Уникальный сквозной ID вопроса
  question: string;        // Текст вопроса
  answers: string[];       // Массив из 4 вариантов ответов
  correctIndexes: number[];// Индексы правильных ответов (0-3)
  page: number;            // Номер страницы в официальных материалах
  category: string;        // Код подкатегории (например, "1a", "1b")
  class: string;           // Категории водительских прав (C/D/CE/DE)
  image: string;           // Путь к изображению (если есть, иначе "")
  explanation: string;     // Пояснение к вопросу (может быть пустым)
}
```

### 2. Логика хранения прогресса (`driver95_progress`)
Вся история ответов пользователя хранится локально под ключом `driver95_progress` в виде сериализованного JSON-объекта:
```typescript
interface ProgressStore {
  [questionId: string]: {
    correct: number;       // Кол-во правильных ответов на этот вопрос
    incorrect: number;     // Кол-во неправильных ответов на этот вопрос
    lastResult: 'correct' | 'incorrect' | null; // Результат последней попытки
    favorite: boolean;     // Добавлен ли вопрос в Избранное
    lastSeen: string;      // ISO строка даты последнего ответа
  }
}
```

### 3. Логика работы PWA и Caching (`service-worker.js`)
- Для кэширования статических ресурсов используется стратегия **Cache-First**.
- При обновлении кода приложения необходимо **обязательно инкрементировать** константу `CACHE_NAME` в `service-worker.js` (например, с `driver95-cache-v13` до `driver95-cache-v14`), иначе браузеры пользователей не увидят изменения.

### 4. Интеграция Google Analytics 4 (`analytics.js`)
- **Идентификатор измерения**: `G-78SW1YDT85`
- Загрузка библиотеки `gtag.js` происходит динамически один раз при инициализации.
- Если аналитика заблокирована ad-blocker'ом, приложение продолжает работу без сбоев. Очередь событий сохраняется в памяти (`analyticsQueue`).
- Отслеживаются события: `app_open`, `language_selected`, `test_started`, `question_answered`, `test_finished`, `test_abandoned`, `pwa_installed`.
- Передаются параметры: `language`, `mode`, `question_id`, `is_correct`, `time_spent`, `app_version`.

## 🎨 Стандарты разработки и стиль кода

### 1. HTML и верстка (`index.html`)
- Использовать семантические теги HTML5 (`<header>`, `<main>`, `<section>`, `<footer>`).
- Все экраны приложения представляют собой блоки `<section class="screen">`, которые скрываются или отображаются переключением класса `.hidden` или `display: none`.
- Кнопка возврата в меню `←` (`#backBtn`) должна быть строго в левом верхнем углу. Стрелки пролистывания вопросов (`#favPrevBtn` / `#favNextBtn`) отображаются справа в шапке в виде символов `‹` и `›`.

### 2. CSS и стилизация (`style.css`)
- Разрешено использовать только чистый CSS (Vanilla CSS).
- Цветовая палитра:
  - Светлая тема: фон `#f8fafc`, текст `#0f172a`, акцент `#2563eb` (синий).
  - Темная тема (класс `body.dark`): фон `#060b13`, блоки `#0d1527`, акцент `#1d63ed`.
- Любой интерактивный элемент при наведении должен иметь плавную анимированную реакцию (`transition: all 0.2s ease`).

### 3. JavaScript и логика (`app.js`)
- Для выбора элементов используется хелпер `$(id) => document.getElementById(id)`.
- **Жёсткое правило**: При удалении или изменении ID элементов в `index.html` вы **ОБЯЗАНЫ** выполнить текстовый поиск по всему файлу `app.js` на наличие старого ID, чтобы избежать падений `TypeError: Cannot set properties of null`.
- Все тексты интерфейса должны быть интернационализированы через объект `I18N`.

### 4. Правила коммитов
Коммиты оформляются по стандарту **Conventional Commits**:
- `feat(progress): ...` — новые фичи
- `fix(quiz): ...` — исправление багов
- `style(theme): ...` — стили, темы
- `docs(agents): ...` — обновление документации

## 🛠️ Инструкции по запуску, тестированию и сборке

### 1. Локальный запуск
Для работы с PWA и симуляции запросов запустите локальный Express сервер:
```bash
# Установка зависимостей сервера
npm install

# Запуск сервера на порту 3000
npm start
```
Приложение откроется на `http://localhost:3000`.

### 2. Сборка
Проект не требует компиляции. Для деплоя достаточно скопировать файлы: `index.html`, `style.css`, `app.js`, `data.js`, `analytics.js`, `manifest.json`, `service-worker.js` и папку `assets/`.

### 3. Тестирование
Перед коммитом изменений обязательно проверьте вручную:
1. Переключение языков на стартовом экране.
2. Сохранение и удаление вопросов из «Избранного» (проверьте, что счетчики на баре и в меню совпадают).
3. Перенос вопроса из «Ошибок» в «Правильные ответы» после успешного решения.
4. Отсутствие ошибок в веб-консоли (F12 / DevTools Console).
